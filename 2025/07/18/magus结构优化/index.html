MAGUS

MAGUS：机器学习和图论辅助通用结构搜索器

功能

  • 为给定的对称性生成原子结构，支持簇、表面、2D 和 3D 晶体，包括分子、受限系统等;
  • 具有 DFT 或主动学习机器学习潜力的大量结构的几何优化;
  • 具有固定或变分成分的结构的多目标搜索;
  • 适用于 VASP、CASTEP、Quantum ESPRESSO、ORCA、MTP、NEP、DeepMD、gulp、lammps、XTB、ASE 等的 API。易于扩展。

4 输入

在MAGUS中，我们使用包括命令行和参数输入文件（yaml格式）在内的输入来控制程序。

如果您是MAGUS的新用户（非常感谢您使用我们的程序！），在阅读下面所有输入的完整描述之前，建议您先查看示例，示例更容易理解。

4.1 输入文件

一个典型的结构搜索任务需要以下输入文件：

  • input.yaml：基本输入文件，定义任务中的主要参数。
  • inputFold：包含计算器所需额外输入文件的文件夹，例如VASP的INCAR和LAMMPS的in.lammps。
  • Seeds：包含种子结构的文件夹。

4.2.1 search

可选参数：

  • -h, –help：显示此帮助信息并退出
  • -ll {DEBUG,INFO,WARNING,ERROR}, –log-level {DEBUG,INFO,WARNING,ERROR}：通过字符串设置详细程度级别：ERROR、WARNING、INFO和DEBUG（默认值：INFO）
  • -lp LOG_PATH, –log-path LOG_PATH：将日志消息保存到磁盘的日志文件（默认值：log.txt）
  • -i INPUT_FILE, –input-file INPUT_FILE：yaml格式的输入参数文件（默认值：input.yaml）
  • -m, –use-ml：使用机器学习加速搜索（默认值：False）
  • -r, –restart：重新开始搜索（默认值：False）
4.2.2 summary

位置参数：

  • filenames：要汇总的文件

可选参数：

  • -h, –help：显示此帮助信息并退出
  • -p PREC, –prec PREC：对称性发现的容差（默认值：0.1）
  • -r, –reverse：是否反向排序（默认值：False）
  • -s, –save：是否保存POSCARS（默认值：False）
  • –need-sort：是否排序（默认值：False）
  • -o OUTDIR, –outdir OUTDIR：保存POSCARS的位置（默认值：.）
  • -n SHOW_NUMBER, –show-number SHOW_NUMBER：在屏幕上显示的数量（默认值：100）
  • -sb SORTED_BY [SORTED_BY …], –sorted-by SORTED_BY [SORTED_BY …]：按哪个参数排序（默认值：Default）
  • -rm REMOVE_FEATURES [REMOVE_FEATURES …], –remove-features REMOVE_FEATURES [REMOVE_FEATURES …]：要从显示特征中移除的特征（默认值：[]）
  • -a ADD_FEATURES [ADD_FEATURES …], –add-features ADD_FEATURES [ADD_FEATURES …]：要添加到显示特征中的特征（默认值：[]）
  • -v, –var：使用可变成分模式（默认值：False）
  • -b BOUNDARY [BOUNDARY …], –boundary BOUNDARY [BOUNDARY …]：在可变成分模式下：添加边界（默认值：[]）
  • -t {bulk,cluster}, –atoms-type {bulk,cluster}

5 常见问题

5.1 如何设置环境变量

MAGUS有两种模式：串行模式和并行模式，由“mode”选项控制。

在串行模式（“mode: serial”）下，只有一个主作业运行。所有子计算（如vasp弛豫）都将在您为主作业申请的计算资源下运行。因此，您可以在主作业中修改环境变量，它们将对所有计算有效。

在并行模式（“mode: parallel”）下，主作业仅用于一些简单计算，并将子作业提交到作业系统以执行更耗时的计算，这意味着子作业采用与主作业不同的环境变量。通常，子作业提交脚本由MAGUS自动生成，但您应该添加一些环境变量以使脚本适合您的系统。最方便的方法是在每个计算器类别下的input.yaml中添加“preprocessing”选项。例如：

magus mutate -h
